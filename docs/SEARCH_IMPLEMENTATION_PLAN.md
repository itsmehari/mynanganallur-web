# Search implementation plan — mynanganallur.in

Multi-layer search and discovery work aligned with the **AI Search Mastery** framework (see [.cursor/skills/ai-search-optimisation-skill-mastery/SKILL.md](../.cursor/skills/ai-search-optimisation-skill-mastery/SKILL.md)): **SEO**, **AEO** (answer surfaces), **GEO** (LLM citability — not “Google Maps”), **AIO** (scalable content systems), **SXO** (conversion), and **Local SEO** (maps, near-me, NAP, local schema).

This document extends [SEO_ENGINE_PLAN.md](./SEO_ENGINE_PLAN.md) with phased tasks, acceptance criteria, and a file index. It does not replace the PRD or execution roadmap.

## 1. Purpose and scope

**Audience:** Engineers implementing routes/schema, editors shaping content structure, operators running GSC/GBP.

**In scope:** Technical SEO, structured data, on-page patterns for answers and citations, local landing quality, content-scale process, basic conversion/analytics hooks documented here.

**Out of scope (listed but not owned in-repo):** Google Business Profile setup, third-party citation campaigns, paid search.

---

## 2. Product and stack snapshot

- **Product:** Neighbourhood-first platform for **Nanganallur** — editorial news, events, jobs, directory-style listings. See [PRD.md](./PRD.md).
- **Stack:** Next.js App Router, Neon Postgres, Drizzle, Vercel. See [README.md](../README.md).
- **SEO principles (from PRD / SEO engine plan):** Canonical URLs, sitemaps, structured data, fast public pages; avoid thin autogenerated hubs without unique copy.

## 5. System flow

Intended chain (from AI Search Mastery): crawlable → understandable → selectable → citable → scalable → convertible → located.

```mermaid
flowchart LR
  crawl[Crawlable_SEO]
  understand[Understandable_SEO]
  select[Selectable_AEO]
  cite[Citable_GEO]
  scale[Scalable_AIO]
  convert[Convertible_SXO]
  locate[Located_LocalSEO]
  crawl --> understand --> select --> cite --> scale --> convert --> locate
```

**Gates:** Rich results and GSC validation are **ops** (run tests after deploy). Editors should fill **`summary`** and optional **`faq_json`** for maximum AEO.

## 6. Phase A — Events: truthfulness and index alignment

**Status: implemented.** Admin **Revalidate public pages** (`src/app/admin/actions.ts`) busts `/` layout cache after DB edits (editor/admin only).

**Goal:** No persistent mismatch between **sitemap/crawl** and **page substance**; enable **Event** rich results when detail pages exist.

**Files:** `src/domains/events/queries.ts`, `src/domains/events/index.ts`, `src/app/(public)/local-events/page.tsx`, `src/app/(public)/local-events/[slug]/page.tsx`, `src/lib/seo/event-jsonld.ts`, `src/app/sitemap.ts`, `src/components/home/home-content.tsx` (`HomeEventsFeatured` async + DB).

**Done**

1. Public events = `status === "scheduled"` (matches seeds). **Index** lists **upcoming** only (`startsAt >= now`). **Detail** resolves any scheduled slug (including past dates).
2. **Event** + **BreadcrumbList** JSON-LD on detail; online vs offline inferred from venue text.
3. Sitemap includes `/local-events/{slug}` via `listEventsForSitemap()`.

**Future:** Per-slug `revalidatePath` when admin CRUD exists.

**Acceptance criteria** (verify in prod)

- [x] `/local-events` DB-backed or empty state.
- [x] Sitemap includes event URLs when rows exist.
- [ ] Event JSON-LD passes Rich Results Test on a live URL.

|----------|-------------|-------------|

- [ ] **Google Search Console:** sitemap submitted (`https://mynanganallur.in/sitemap.xml`), monitor coverage and “crawled — currently not indexed” on thin URLs.
- [ ] **Canonical + sitemap:** `loc` matches internal links and `alternates.canonical` for sampled URLs.
- [ ] **Manual:** First screen answers one clear user question; at least one **citable** sentence for LLM-style extraction on pillar pages.
- [ ] **GBP / NAP** (ops): consistent only if a real business location applies.

---

## 13. Mistakes to avoid

- FAQPage (or any schema) **without** matching visible content.
- **Thin** area or hub pages **in the sitemap** while claiming editorial quality.
- **Event** / **JobPosting** JSON-LD with missing required properties.
- **Invented or inconsistent NAP** for LocalBusiness.
- Relying on **AI-generated drafts** without human fact-check for names, dates, and legal/safety claims.
