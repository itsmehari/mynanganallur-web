# SEO engine plan

## Lessons applied from legacy

1. **Canonical URL** must match sitemap `loc` and internal links.
2. **One metadata pipeline** — Next.js `generateMetadata` per route; no duplicate `<title>` sources.
3. **Structured data:**
   - `Organization` + `WebSite` on home
   - `NewsArticle` or `Article` for news
   - `Event` for events
   - `JobPosting` for jobs (when fields complete)
   - `BreadcrumbList` when hierarchy exists
4. **Avoid invalid schema** — conditional blocks only when required fields exist (legacy used layered checks for sports; prefer **data-driven** schema builders).
5. **GA4:** measurement ID via env; separate numeric property ID for Data API — never confuse the two.

## Implementation

- **ISR / caching:** Use `revalidate` on public pages; `revalidatePath` / `revalidateTag` on publish.
- **Sitemaps:** `src/app/sitemap.ts` (main inventory), `sitemap-news.xml`, `sitemap-images.xml`; referenced from `robots.ts`. Exclude `noindex` routes (`/search`, `/submit/thanks`, `/my/*`).
- **Canonical URLs:** `src/lib/seo/canonical-url.ts` + `buildHubMetadata` / `buildPageMetadata`. Sitemap `loc` must match `alternates.canonical` and internal links. Directory verticals: `/directory/{type}` (path-based, not query strings).
- **Host normalization:** `www.mynanganallur.in` → apex via `next.config.ts` redirect.
- **Search Console:** Domain property verified; submit/resubmit sitemap after URL inventory changes. See [SEARCH_CONSOLE.md](./SEARCH_CONSOLE.md).

## Anti-patterns to avoid

- Default meta keywords stuffed with legacy locality names.
- Thin autogenerated hub pages without unique copy.
